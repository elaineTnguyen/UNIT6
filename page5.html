6.7 - Cryptography: Securing The Internet

1) Explain in your own words how public key encryption is not symmetric.

Public key is encryption is not symmetric because the encrypted message is sent to the reciepient with one key, the reciepient will then add their own key (lock) onto the encrypted message and send it back to the original owner. The original owner will then proceed to take off their lock and send the message to the reciever to finally read the message. As this whole process continues the key is never shared with one another, rather the same lock you would put on the message would only be able to be broken by you.
